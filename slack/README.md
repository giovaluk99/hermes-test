# Hermes Slack app (talk to the agent from Slack)

`manifest.json` is generated by `hermes slack manifest` — it registers all 50
gateway commands as Slack slash commands (`/hermes`, `/btw`, `/new`, …) and is
configured for **Socket Mode** (`socket_mode_enabled: true`), so **no public URL
is needed** (the placeholder `hermes-agent.local` slash URLs are ignored — Slack
delivers events/commands over an outbound WebSocket).

This is the *channel* integration (you DM/mention the bot, Hermes replies) —
distinct from the Composio/bot-token *tool* access. Use a **dedicated** Slack app
(not the shared `coworker` bot) so identities don't collide.

## Setup (your steps)
1. https://api.slack.com/apps → **Create New App → From a manifest** → pick your
   workspace → paste `manifest.json` → Create.
2. **Basic Information → App-Level Tokens** → Generate Token, scope
   `connections:write` → copy the `xapp-…` token.
3. **Install App** to the workspace → copy the **Bot User OAuth Token** `xoxb-…`.
4. On the instance, give Hermes both tokens and start the Slack channel, e.g.:
   ```bash
   # in the instance (web terminal :7681, or via exec)
   printf 'SLACK_BOT_TOKEN=xoxb-...\nSLACK_APP_TOKEN=xapp-...\n' >> ~/.hermes/.env
   hermes slack            # or: hermes gateway restart, depending on build
   ```
   (Hermes reads `~/.hermes/.env` at gateway start; Socket Mode connects out.)
5. Invite the bot to a channel and `/hermes hello` or @mention it.

Regenerate after a Hermes upgrade (new commands):
`hermes slack manifest --name "Hermes (Agent37)" --write` then re-paste under
**Features → App Manifest**.
